Input from user Module

This node interacts with the user to control a robot’s movement inside a virtual arena. The user is prompted to enter coordinates (x, y) for the robot to reach. The node sends these coordinates as goals to an action server. The node also subscribes to the /odom topic to receive the robot’s odometry information, and publishes the robot’s position and velocity on the /Position_velocity topic. The user can choose to stop the robot by entering ‘S’.

scripts.input_from_user.client()[source]
Interacts with the user and sends goals to the action server.

This function creates an action client for the /reaching_goal action server and waits for it to start. It then enters a loop where it prompts the user to enter the coordinates (x, y) that the robot must reach. If the input is valid, it creates a goal message with the target coordinates and sends it to the action server. The user can also choose to stop the robot by entering ‘S’ and pressing Enter.

scripts.input_from_user.publish_values(msg)[source]
Publishes the position and velocity values extracted from an Odometry message.

This function receives an Odometry message and extracts the position (x, y) and velocity (linear x, linear y) information. It creates a custom message of type Pos_vel and publishes it on the /Position_velocity topic.

Information Module

This node subscribes to the /Position_velocity topic and prints information about the robot’s position and velocity. It computes the distance between the desired position (set by the user) and the actual position of the robot, as well as the average speed of the robot. The information is printed at a specified frequency.

scripts.information.main()[source]
Main entry point of the ROS information node.

This function initializes the ROS node, gets the publishing frequency from the parameter server, and subscribes to the /Position_velocity topic to receive position and velocity information. It enters a spin loop to continue receiving messages until the node is shut down.

Service Module

This node provides a service and subscribes to the result topic to keep track of the number of goals cancelled and goals reached. It implements a service called ‘service’ that returns the current count of cancelled and reached goals.
